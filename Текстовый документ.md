2. Общее описание системы
2.1. Назначение системы

Веб-платформа предназначена для каталогизации, безопасной продажи и покупки цифровых игровых аккаунтов.

Система автоматизирует:

    Листинг товаров

    Обработку платежей

    Мгновенную выдачу учётных данных после оплаты

    Арбитраж сделок

    Техническую поддержку

2.2. Целевая аудитория
Роль	Описание
Покупатели	Пользователи, ищущие игровые аккаунты с определёнными играми, прогрессом или региональными ограничениями
Продавцы	Частные лица или небольшие команды, выставляющие аккаунты на продажу
Администраторы / Модераторы	Персонал платформы, отвечающий за проверку листингов, разрешение споров, управление пользователями и контентом
2.3. Основные сценарии использования

    Регистрация и верификация пользователя

    Просмотр каталога, применение фильтров (цена, игры, регион, рейтинг продавца)

    Добавление товара в корзину / избранное

    Оформление заказа и оплата через интегрированный платёжный шлюз

    Автоматическая выдача данных аккаунта (логин / пароль) после подтверждения платежа

    Управление личным кабинетом (история заказов, баланс, выставленные товары, тикеты)

    Арбитраж — создание обращения в поддержку при проблемах с доступом к купленному аккаунту

    Модерация — проверка новых листингов, блокировка мошеннических аккаунтов, управление пользователями

5. Краткое описание ключевых технологий
TypeScript

Используется вместо обычного JavaScript для строгой типизации данных. Это позволяет обнаруживать ошибки в коде ещё на этапе написания (например, если вы пытаетесь передать текст туда, где ожидается число), а также улучшает автодополнение кода в редакторе.
Next.js

Основной инструмент для построения интерфейса. React отвечает за создание интерактивных компонентов (кнопок, форм, карточек товаров). Next.js надстраивается над React и предоставляет готовую маршрутизацию, серверный рендеринг (для быстрой загрузки и SEO) и встроенные возможности для оптимизации изображений и шрифтов.
Tailwind CSS

Библиотека для вёрстки, которая позволяет задавать стили прямо в HTML-классах (например, class="text-red-500 p-4"). Это избавляет от необходимости создавать отдельные CSS-файлы для каждого компонента, ускоряет процесс вёрстки и гарантирует единообразный дизайн.
Zustand

Лёгкая библиотека для хранения глобальных данных приложения. Используется для того, чтобы информация о пользователе (залогинен ли он) или содержимое корзины были доступны на любой странице сайта без необходимости передавать их через все компоненты подряд.
React Hook Form

Инструмент для создания сложных форм (регистрация, оформление заказа). Эффективно управляет полями ввода, не перерисовывая всю страницу при каждом нажатии клавиши.
TanStack Query

Библиотека, которая упрощает получение данных с бэкенда. Автоматически кэширует ответы сервера, показывает статусы загрузки (спиннеры) и обрабатывает ошибки сети, избавляя разработчика от написания большого количества шаблонного кода.
Next.js API Routes

Бэкенд реализован внутри того же проекта Next.js. Специальные файлы работают как серверные обработчики запросов. Когда фронтенд отправляет данные, они попадают в эти роуты, где происходит проверка прав доступа, валидация и обращение к базе данных.
PostgreSQL

Надёжная реляционная система управления базами данных. Отлично подходит для хранения структурированной информации с чёткими связями: один пользователь может иметь много заказов, один заказ содержит много товаров и т.д. Гарантирует целостность данных даже при сбоях.
Prisma

Инструмент-посредник между кодом на TypeScript и базой данных PostgreSQL. Разработчик описывает структуру таблиц в одном файле схемы, а Prisma автоматически генерирует типы TypeScript и функции для запросов. Это делает работу с базой данных безопасной.
JWT + Cookies

Механизм входа в систему. После успешного ввода логина и пароля сервер создаёт зашифрованный токен (JWT), который подтверждает личность пользователя. Токен сохраняется в защищённой куке браузера.
Bcryptjs

Библиотека для хеширования паролей. Пароли пользователей никогда не хранятся в открытом виде. Перед записью в базу данных пароль превращается в сложную строку символов (хеш). Даже если базу данных украдут, злоумышленники не смогут узнать реальные пароли.
Redis

Сверхбыстрое хранилище данных в оперативной памяти. Используется для временного хранения часто запрашиваемой информации (например, списков популярных товаров) или чёрных списков заблокированных токенов. Это снижает нагрузку на основную базу данных PostgreSQL и ускоряет ответ сервера.
